# Add Note — Quick Thought Capture

You are capturing a single thought for the vault owner. This should be fast — under a minute from start to finish.

## The flow

### Step 1 — Ask what they want to capture
"What's the thought?"

### Step 2 — Figure out where it goes
Options (pick the best fit):
- **Append to an existing wiki page** — if the thought relates to something already captured
- **Create a new wiki page** — if it's a new topic worth its own file
- **Add to the pillar file** — if it's about the business at a high level
- **Drop into inbox/** — if you're not sure where it belongs yet

Tell the owner: "I'd add this to [[wiki/page-name]]. Sound right?"

### Step 3 — One clarifying question (if needed)
If the thought is ambiguous, ask ONE question. Not two. Not three. One.

If it's clear enough, skip this step entirely.

### Step 4 — Show your work, then write
Show the exact text you'd add and where. Format:
```
- YYYY-MM-DD — "[their exact words]"
```

Once approved, write it.

### Step 5 — Confirm and ask if there's more
"Done. Anything else, or are we good?"

## Rules

- **Append, never overwrite.** Add to existing content, don't replace it.
- **Date-stamp everything:** `- 2026-04-14 — "the thought"`
- **Use their exact words.** No paraphrasing. If they said "I think my pricing is too low," write exactly that.
- **Link to related notes** with `[[double brackets]]` where relevant.
- **Use the file schemas from CLAUDE.md** if creating a new file.

## When to bail

If their "one thought" turns into 3+ paragraphs, say:
"This sounds like it deserves a proper interview. Want to switch to /interview instead?"
